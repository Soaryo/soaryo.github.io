面向对象的JS

鸭子类型

只关注对象的行为，而不关注对象的本身。一只动物能否发出声音取决于它有没有makesound()方法，而不在于它是某种类型的对象，不存在“类型耦合”

多态

多态：同一操作在不同对象上，可产生不同的解释和不同的执行结果。

对象的多态性
多态背后的思想:”将做什么”和”谁去做及怎样做”分离，就是将”不变的事物”和”可能改变的事物”分离

多态的作用：通过把过程化的分支语句转化为对象的多态性，从而消除条件分支语句

封装

封装的目的：将信息隐藏，包括封装数据、封装实现、封装类型、封装变化。
封装数据：js中依赖变量作用域来实现封装，只能模拟出public和private两种封装性

原型继承

  1. 所有的数据都是对象
  2. 要得到一个对象，不是通过实例化，而是找到一个对象作为原型并克隆它
    调用var obj =new Object()引擎内部会从Object.prototype上克隆一个对象
  3. 对象会记住原型
  4. 如果对象无法响应某个请求，它会把这个请求委托给它自己的原型

this、call、和apply

this

this指向

  1. 作对象方法调用

  2. 作普通函数调用
    this总是指向全局对象

  3. 构造器调用
    通常情况下：构造器中的this指向返回的这个对象

  4. Function.prototype.call and Function.prototype.apply
    call()和apply()可以动态改变传入函数的this

call和apply的用途

  1. 改变this指向

  2. Function.prototype.bind

  3. 借用对象方法

闭包和高阶函数

闭包

  1. 变量作用域：变量的有效范围，函数可以创建作用域
  2. 变量的生存周期：全局变量的生存周期永久，除非主动销毁。函数内的局部变量，随函数的调用的结束而销毁。
  3. 如果函数返回局部变量的引用，那么局部变量就不会被销毁，形成闭包
